ContextSMS Webex API (1.0.0)

Download OpenAPI specification:

Welcome to the ContextSMS Webex API Documentation. This guide explains how to use the ContextSMS APIs in a simple and organized way. It shows you the available endpoints, how to authenticate, what parameters you need, and how the API functions.

The information is written to help both technical teams and developers use the API effectively. Whether you are creating new messaging features or improving current systems, this documentation gives you all the details needed to connect successfully.

You will find easy-to-understand explanations, standard formats for requests and responses, and examples to help you build and troubleshoot your integration. The ContextSMS Webex API is designed to be easy to use, so you can implement it quickly and keep it running smoothly.

Note: Parameters with an asterisk (*) are required when making API calls.

For a step-by-step introduction, see the Getting Started Tutorial.

Mapping Management

APIs for creating, retrieving and deleting number-to-message-app mappings.

Add Mapping

Create a new mapping for an enterprise.

Notes:

  • messagingAppId is case-sensitive and must match the email in Webex.
  • mmsEnabled must follow these rules:
    • If MMS feature is Disabled → mmsEnabled: false
    • If MMS feature is Strict → mmsEnabled: true
    • If MMS feature is Partial → both true/false allowed

enterpriseId

string
